### 함수 표현식 vs 함수 선언식

### 1. 함수 선언식 - Function Declarations

```jsx
function 함수() {
  //
}
함수();
```

### 2. 함수 표현식 - Function Expressions

- 함수 표현식은 변수로 저장될 수 있다.
- 변수에 저장된 함수는 함수명이 필요 없으며, 변수 이름을 통하여 호출된다.

```jsx
const 함수 = function () {
  //
};

함수();
```

### 3. 함수 선언식과 표현식의 차이점

- 함수 선언식은 호이스팅에 영향을 받지만, 함수 표현식은 호이스팅에 영향을 받지 않는다.
- 호이스팅? : JS 엔진이 자바스크립트 코드를 해석할 때 맨 위로 끌어 올려지는 현상.

```jsx
함수A();
함수B();

// 함수선언식
function 함수A() {
	return 'hi';
}

// 함수표현식
const 함수B = function() {
	return 'Bye";
}

```

### 함수 선언식의 호이스팅

- 이떄 함수선언식으로 선언한 `함수A` 는 호이스팅되어 먼저 선언된다.
- 함수 전체를 호이스팅한다.
- 정의된 범위의 맨 위로 호이스팅되서 함수 선언 전에 함수를 사용할 수 있다.
- 따라서 `함수A();` 는 실행됨

### 함수 표현식과 호이스팅

- 그러나 함수표현식으로 선언한 `함수B` 는 호이스팅의 영향을 받지 않기때문에, 선언문 이전에 함수를 실행해면
- 함수B is not a function 에러가 발생한다.
- 변수는 선언부와 할당부를 나누어 호이스팅하게 된다. **선언부만 호이스팅**된다.
